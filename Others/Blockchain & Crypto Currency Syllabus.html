Course:

Blockchain & Cryptocurrency Trading (prerequisite:  Python syntax, functions, loops, importing libraries, and constructing classes)

Course description:

The exchange of Bitcoin, a digital currency that has drastically appreciated in value over the years, has been made possible by an underlying technology known as blockchain, which not only secures and simplifies transactions, but also eliminates the need for a bank or central authority, giving anyone with an internet connection the freedom to own and exchange this cryptocurrency. As businesses have begun to recognize the potential of blockchain, they are rapidly mobilizing to comprehend and implement it. Therefore, it is essential to understand what blockchain is and what makes it so transformative. Upon completion of this course, you will have gained a basic foundation of blockchain principles, as well as the opportunity to create your very own mini blockchain by transforming these concepts into code.

  • What is Blockchain?
  • Lesson Description:
  • Aim to describe: Blockchain is a permanent and accurate record of all transactions in chronological order, similar to a book of records. It does not rely on a central authority like a bank, but instead uses a public network of computers to verify transactions. Each block represents a transaction and is linked chronologically to form the entire blockchain.
  • The Blockchain Network
  • Lesson Description:
  • Aim to describe: Blockchain-based applications like Bitcoin and Ethereum use a decentralized network of participants who own a copy of the blockchain and verify transactions, eliminating the need for a central authority. To add a new block, 51% of participants must verify its validity, and once verified, it is added to all copies of the blockchain, reducing transaction fees and ensuring security.
  • What are Blocks in the Blockchain?
  • Lesson Description:
  • Aim to describe: Blocks are the building blocks of a blockchain, which contain transaction data, a timestamp, a unique hash, and a reference to the prior block's hash. These attributes make a block part of a blockchain structure, and the previous hash is what makes the blockchain unique, as it creates a chain of blocks. The timestamp determines the location of the block on the blockchain, and the hash is a unique code that combines all the contents within the block, acting as a digital fingerprint.
  • Hashing
  • Lesson Description:
  • Aim to describe: Hashing is a cryptography technique used in blockchain that generates a seemingly random but calculated string of letters and numbers from any input by using a hash function. The output, known as the hash, is deterministic, meaning the same input will always produce the same hash, but the hash cannot be used to produce the original input.
  • The Genesis Block
  • Lesson Description:
  • Aim to describe: A blockchain is a permanent record of data stored in chronological order. Each block has a reference to the previous block's hash, creating a chain that prevents tampering. The first block, known as the Genesis Block, does not have a previous hash, so its value is typically hard-coded as zero.

  • Gathering Blockchain Transactions
  • Lesson Description:
  • Aim to describe: Blockchain transactions work, including the placement of unvalidated transactions in the mempool and the process of collecting transactions into a new block. Each block has a limited capacity, and once it is full, the next set of transactions will wait in the mempool until the block is confirmed. The text also mentions that the blockchain technology can be customized to fit different applications.
  • Adding More Blocks
  • Lesson Description:
  • Aim to describe: The process of how blocks are confirmed and added to the blockchain. It involves verifying transactions and establishing consensus among network participants. Once a block is broadcasted and more than 51% of the network agrees with it, the block is confirmed. The blockchain remains secure as long as the majority of participants verify transactions honestly. Consensus is the process of agreeing to the transactions on the blockchain network.
  • How Hashing Maintains the Blockchain's Integrity
  • Lesson Description:
  • Aim to describe: The use of hashing is fundamental to the design of the blockchain because it creates a unique and immutable record of transactions. Each block in the blockchain has a unique hash that links it to the previous block, and any attempt to tamper with a block would result in a mismatch between hashes and break the link between blocks, resulting in an invalid copy of the blockchain. This makes the records in the blockchain immutable, meaning they cannot be altered. The next exercise explores whether hashing still guarantees full security if someone tampers with a block and recalculates the hashes for every block.
  • Is Hashing Enough to Secure the Blockchain?
  • Lesson Description:
  • Aim to describe: The possibility of an attacker tampering with a block in a blockchain and covering their tracks by recalculating the hash of subsequent blocks. To illustrate this concept, the text provides an example of an attacker changing the contents of Block A, resulting in changes to the subsequent blocks' hashes. The only way to make the blockchain valid again is to recalculate the hashes of subsequent blocks, which also counts as tampering with their data. The attacker can repeat this process to create a valid copy of the blockchain.
  • Securing the Blockchain Further
  • Lesson Description:
  • Aim to describe: The security measures in previous exercises are not enough to fully secure the blockchain from outside interference. Proof-of-Work is a technique used to make it difficult for someone to tamper with subsequent blocks and create a trustless system. It relies on cryptography to verify transactions, and is the main reason why the blockchain is secure and powerful.
  • Proof-of-Work
  • Lesson Description:
  • Aim to describe: The blockchain network relies on a computationally difficult math problem, known as Proof-of-Work, to verify transactions. Miners, a special group of participants, solve the problem by guessing a nonce value and combining it with the contents of the block until the correct hash is generated. The first miner to produce a correct hash broadcasts their unconfirmed block and the network verifies the calculation. The key terms include miners and nonce.
  • Diving Deeper into Proof-of-Work
  • Lesson Description:
  • Aim to describe: The blockchain network considers the longest chain to be the correct one since it assumes that it has the most proven record of transactions due to the amount of computational work done in finding the Proof-of-Work for each block. It is computationally infeasible to tamper with a block as it would require solving the Proof-of-Work for each subsequent block, and newer blocks are added at a faster rate. The key takeaway is that newer blocks added to the blockchain make it increasingly tamper-proof, and Proof-of-Work makes it difficult to introduce fake transactions.
